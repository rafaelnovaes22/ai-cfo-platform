# Case hub-0027 — Isolamento: 2 tenants com mesmo referenceMonth

## Input
- DB tenant_A: [{ id:"an_a", referenceMonth:"2026-04", costCents:1000, actionPlanJson:{totalImpact:111111} }]
- DB tenant_B: [{ id:"an_b", referenceMonth:"2026-04", costCents:9999, actionPlanJson:{totalImpact:222222} }]
- request: GET /analyses (Bearer token tenant=tenant_A)

## Ground truth
```json
{
  "analyses": [
    {
      "id": "an_a",
      "referenceMonth": "2026-04",
      "costCents": 1000,
      "totalImpactCents": 111111
    }
  ]
}
```

## Justificativa
R11: dois tenants com mesmo `referenceMonth="2026-04"` não interferem. Filtro `where:{tenantId}` isola. Análise de tenant_B (an_b) jamais aparece em GET /analyses de tenant_A.
