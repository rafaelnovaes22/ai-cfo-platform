# Case hub-0032 — Query params espúrios ignorados (limit/cursor/offset)

## Input
- subscription: { plan: "essencial", mode: "assisted", status: "active" }
- monthlyAnalyses: 15 análises
- request: GET /analyses?limit=100&cursor=fake&offset=20&orderBy=asc

## Ground truth
```json
{
  "analyses_length": 12,
  "first_referenceMonth": "{mais recente}",
  "prisma_args": {
    "take": 12,
    "skip": "undefined",
    "cursor": "undefined",
    "orderBy": { "referenceMonth": "desc" }
  }
}
```

## Justificativa
R9 negativo: backend ignora `limit`, `cursor`, `offset`, `orderBy` da query. Comportamento v1 fixo: `take:12`, sem skip/cursor, `orderBy desc`. Defesa contra cliente que tenta paginar manualmente.

## Tags
adversarial, query-param-injection, v1-fixed-pagination
