# Case hub-0012 — Cross-tenant: ?tenantId=tenant_A com JWT de tenant_B

## Input
- subscription tenant_A: { plan: "growth", mode: "autonomous", status: "active" }, com 3 análises ready
- subscription tenant_B: null (tenant novo)
- request: GET /hub?tenantId=tenant_A (Bearer token tenant=tenant_B)

## Ground truth
```json
{
  "subscription": { "plan": "trial", "mode": "shadow", "status": "active" },
  "latestAnalysis": null
}
```

## Justificativa
EC10: query param `?tenantId=` é ignorado. Backend usa `req.auth.tenantId` (=tenant_B). Resultado: subscription default + análise null (tenant_B é novo). Não vaza dados de tenant_A.
