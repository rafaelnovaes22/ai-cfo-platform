# Case hub-0026 — Cross-tenant attempt: ?tenantId=tenant_A com JWT tenant_B

## Input
- DB: tenant_A tem 5 análises ready; tenant_B é novo (0 análises)
- request: GET /analyses?tenantId=tenant_A (Bearer token tenant=tenant_B)

## Ground truth
```json
{
  "analyses": []
}
```

## Justificativa
R11 negativo: query param `?tenantId=` é ignorado. `findMany` usa `where: { tenantId: tenant_B }` do JWT. Resposta: lista vazia (tenant_B é novo). Não vaza nenhuma das 5 análises de tenant_A.
