# Case hub-0025 — Análise antiga sem campos novos (backward compat)

## Input
- subscription: { plan: "essencial", mode: "assisted", status: "active" }
- monthlyAnalyses: [{ id:"an_old", referenceMonth:"2025-08", status:"delivered", mode:"assisted", deliveredAt:"2025-09-02T10:00:00Z", approvedAt:null, costCents:null, actionPlanJson:null }]
- request: GET /analyses

## Ground truth
```json
{
  "analyses": [
    {
      "id": "an_old",
      "referenceMonth": "2025-08",
      "status": "delivered",
      "mode": "assisted",
      "deliveredAt": "2025-09-02T10:00:00.000Z",
      "approvedAt": null,
      "costCents": null,
      "totalImpactCents": null
    }
  ]
}
```

## Justificativa
Compatibilidade backward: análises antigas (pré-Onda1) podem não ter `costCents`/`actionPlanJson` populados. Backend serializa todos como null sem quebrar schema.
