# Case hub-0018 — Tentativa de override via body POST em rota GET

## Input
- subscription: { plan: "essencial", mode: "shadow", status: "active" }
- monthlyAnalyses: 1 análise ready
- request: GET /hub com body `{ "tenantId": "tenant_outro", "mode": "autonomous" }` (cliente tenta forçar mode autonomous para burlar SHADOW redaction)

## Ground truth
```json
{
  "subscription": { "plan": "essencial", "mode": "shadow", "status": "active" },
  "latestAnalysis": {
    "dre": null,
    "cards": { "critical_gap": 0, "attention": 0, "healthy": 0 },
    "actionPlan": null
  }
}
```

## Justificativa
GET ignora body. `subscription.mode` lido do DB (=shadow). Redaction backend-side (f62d508) persiste mesmo com tentativa de injeção. Defesa contra bypass cliente-side.
